# 📘 PART 01. ChatGPT: 생성형 AI와 거대 언어 모델(LLM)의 이해

### 💡 생성형 AI (Generative AI)

* **개념:** 기존 데이터를 학습하여 텍스트, 이미지, 음악, 비디오 등 **새로운 콘텐츠를 창조하는 기술**


* **주요 특징:**  주어진 입력(프롬프트)에 따라 맞춤형 글 작성 및 스타일 이미지 생성


* 예술, 디자인, 마케팅, 게임, 교육 등 다양한 산업에서 창의적인 아이디어를 제공하는 도구로 활용



### 🤖 거대 언어 모델 (LLM: Large Language Model)

* **개념:** 자연어 처리(NLP) 분야에 특화된 생성형 AI의 한 유형으로, 인간처럼 언어를 이해하고 생성하는 데 주력


* **주요 특징:**
* **언어 이해:** 문맥을 파악하고 문법적으로 올바른 문장을 생성 (대화형 AI, 번역, 요약 등)


* **대화 생성:** 고객 서비스 챗봇이나 개인 비서처럼 자연스럽고 일관된 대화 유지가 가능


* **지식 기반:** 방대한 데이터를 바탕으로 특정 주제에 대한 고도화된 답변 제공





### 🌐 주요 글로벌 LLM 서비스 현황

* **ChatGPT (OpenAI):** GPT 모델 기반, 압도적인 성능과 친숙한 채팅 인터페이스로 대중화를 선도


* **Gemini (구글):** 텍스트, 이미지, 오디오, 비디오를 동시에 처리하는 멀티모달 모델


* **LLaMA (메타):** 연구 및 개발을 위해 자유롭게 활용 가능한 오픈 소스 모델


* **Claude (앤트로픽):** 윤리적이고 안전한 AI 개발 및 유해 콘텐츠 최소화 목적 설계


* **DeepSeek (중국):** 코딩 및 수학적 추론에 특화된 고성능 모델


* **Grok (xAI):** 소셜 미디어(X) 데이터 기반의 실시간 분석과 유머러스한 답변 제공


* **국내 특화 모델:** 한국어 데이터 및 콘텐츠 생성에 강한 **CLOVA-X(네이버)** 및 AI 포털 **뤼튼(wrtn)**

> 
> 

* **설정 요소:**
1. AI가 사용자를 어떻게 부르면 좋은지 (예: 소희) 
2. 사용자가 어떤 일을 하고 있는지 (예: 인공지능 소프트웨어학과 3년제 학생)

## 4. OpenAI의 혁신과 GPT의 진화 과정

### 📈 핵심 기술 용어

* **GPT (Generative Pre-trained Transformer):** 생성 사전 훈련 변환기. 인터넷의 문장 구조를 학습해 자연스러운 다음 단어를 예측하는 능력을 갖춘 글쓰기 모델
* **사전 훈련 (Pre-training):** 방대한 인터넷 텍스트 데이터를 읽어 단어의 의미와 문법, 세상의 지식을 익히는 기본 학습 과정


* **미세 조정 (Fine-tuning):** 사전 훈련된 모델에 특정 분야의 데이터를 추가 수업하여 시인, 번역가 등 특화된 전문가로 만드는 과정


* **RLHF (인간 피드백 기반 강화 학습):** ChatGPT 1.0부터 본격 도입되었으며, 인간의 피드백을 통해 모델의 유해성을 줄이고 대화 생산성을 최적화하는 기술

# 📘 PART 02. ChatGPT: 대화 질문 설계 기초 다지기 (프롬프트 엔지니어링)

### 1. 대화 질문 설계(프롬프트 엔지니어링)의 정의와 목적

* **정의:** 생성형 AI와의 대화에서 원하는 결과를 얻기 위해 질문을 구조화하고 최적화하는 기술 및 과정
* **5대 목표:** 질문 설계, 최적화 기술, 사용자 의도 전달, AI 활용, 성능 극대화



### 2. 단계별 구체적 질문 기법: `Let's think step by step`

* **개념:** AI에게 "단계별로 생각해서"라는 지시어를 추가하는 접근 방식
* **효과:** 복잡한 문제(예: 환경 문제 해결, 서울 여행 일정 짜기)를 단순한 단계(문제 정의 $\rightarrow$ 원인 분석 $\rightarrow$ 해결책 제안 등)로 나누어 답변의 정확도와 세부 일정의 구체성을 비약적으로 향상



### 3. ⭐ [필수 암기] 대화 질문 설계의 10가지 구성 요소


| 구성 요소 | 설명 | 예시|
| --- | --- | --- |
| **목적 (Objective)** | 글이나 요청의 목적을 명확히 설정| "전기차 구매 시 고려사항을 알려줘"|
| **상황 (Context)** | 배경 정보, 상황, 제약 사항 제공| "60세 이상 남성을 위한 운동"|
| **역할 (Role/Persona)** | ChatGPT에게 특정 역할이나 관점 부여| "너는 10년 경력의 전문 요리사야"|
| **청중 (Audience)** | 답변을 읽을 목표 독자를 구체적으로 명시| "초등학교 5학년 수준", "대학 신입생"|
| **예시 (Examples)** | 원하는 형식이나 스타일의 샘플 데이터 제공| "'긴 글 $\rightarrow$ 핵심 요약' 형태로 만들어줘"|
| **형식 (Format)** | 결과물의 구조나 아웃풋 형태 지정| "표 형식으로 정리해줘", "문단으로 써줘"|
| **키워드 (Keywords)** | 반드시 포함해야 할 핵심 단어나 개념 지정| "글루텐 프리", "반복적 접근 포함"|
| **제한 (Limitations)** | 답변의 길이, 단어 수, 문장 수 제약 설정| "200자 이내", "각 운동은 30분 이내"|
| **어조 (Tone)** | 글의 분위기와 스타일 지정| "친절하게 설명해줘", "전문가 시각에서"|
| **부정 (Negation)** | 답변에서 제외하거나 피해야 할 내용 명시| "무릎에 부담을 주는 운동은 제외해줘"|

### 4. 마크다운(Markdown) 및 특수 문자 기호 활용

* **샵 기호 (`#`):** 마크다운 문법에서 제목이나 섹션(서론, 본론, 결론)을 구분하여 구조화


* **세 개의 백틱 (`````):** 프롬프트 내에서 코드 블록(Python, JSON, SQL)이나 명령어 구분


* **기타:** 목록 작성용 하이픈(`-`), 강조용 따옴표(`""`) 및 별표(`*`), 추가 조건용 대괄호(`[]`)와 중괄호(`{}`)



### 5. 퓨샷 프롬프팅 (Few-shot Prompting)



* **퓨샷 러닝 (Few-shot Learning):** 제한된 수(단 몇 개)의 예시만으로 새로운 개념이나 작업을 신속하게 학습하는 기술


* **프롬프팅 유형 비교:**
* **제로샷 (Zero-shot):** 아무런 예시 없이 바로 결과 요청 (예: "에펠탑을 설명해줘")


* **원샷 (One-shot):** **1개**의 질문-답변 예시 제공, 패턴 유도


* **투샷 (Two-shot):** **2개**의 예시 제공


* **퓨샷 (Few-shot):** **여러 개**의 구체적인 예시 제공, 복잡한 규칙이나 감정 분류(긍정/부정)의 정확도를 극대화

# 📗 PART 04. ChatGPT 앱과 나만의 맞춤형 GPT 만들기

### 1. ChatGPT 데스크탑 독립 실행형 앱의 장점



* 웹 브라우저 없이 독립 실행 가능, 쾌적한 멀티태스킹 환경 제공


* 단축키(`Alt + Tab` 전환 등), 다크 모드, 시스템 알림 등 네이티브 기능 연동


* 브라우저 대비 메모리 사용량이 감소하여 실행 안정성 및 업무 효율성 최적화



### 2. 개인 맞춤형 GPT (내 GPT / GPTs) 개요



* **개념:** OpenAI가 유료 플랜 사용자에게 제공하는 기능으로, 코딩 지식 없이 프롬프트 지침과 지식 문서 설정만으로 특정 목적에 최적화된 **나만의 맞춤형 챗봇**을 만드는 기능


* **조정 가능 영역:** 대화 톤 조정(말투), 응답 길이 조정, 주제 범위 조정(지식 확장), 학생용/직장용 맞춤 도우미 기능



### 3. '내 GPT' 만들기 핵심 구성 요소



* **만들기 (Create) 탭:** 대화 형식으로 GPT가 가질 전문성과 역할, 목적을 자연어로 작성하는 공간


* **구성 (Configure) 탭 (필수 세부 항목):**
* **이름 (Name) & 설명 (Description):** GPT의 명칭과 명사형 역할을 명확히 입력


* **지침 (Instructions):** AI가 지켜야 할 아웃풋 구조, 타겟 청중 수준(예: 컴공 2~3학년 수준), 기본값 규칙 등을 세부 지정


* **대화 스타터 (Conversation Starters):** 사용자가 쉽게 대화를 시작할 수 있도록 돕는 예시 질문 버튼


* **지식 (Knowledge):** **'GPT의 뇌에 책을 넣는 과정'**. 참조할 문서, 교재, PDF 파일 등을 업로드하여 해당 데이터 기반으로 답변하도록 설정


* **기능 (Capabilities):** 웹 검색, DALL-E 이미지 생성, 코드 인터프리터(데이터 분석) 활성화 여부 선택


* **작업 (Actions):** 외부 API 연동을 통해 파워포인트 자동 생성이나 외부 데이터 접속 자동화 설정





### 4. 실무 응용 사례 및 폰트 문제 해결



* **'SlidePro for Computer Science' 구축:** 각 슬라이드별 제목, 소제목(3~5개), 명사구 형태의 설명 문단(200~300자), Python 코드 블록 및 라인별 설명을 구조적으로 출력하도록 지침 설계


* **PDF 한글 깨짐 방지 대책:** ChatGPT에서 한글이 포함된 PDF 파일 생성 시 글꼴 문제로 글자가 깨지는 현상을 막기 위해, '지식(Knowledge)' 영역에 **실제 한글 폰트 파일(`.ttf` 또는 `.whl`)을 직접 업로드**, 지침에 해당 폰트를 사용하도록 명시


* **공유 및 업데이트:** 완성된 GPT는 우측 상단 업데이트 버튼을 통해 공개 범위(나만 보기, 링크가 있는 사람, GPT 스토어) 선택 배포 가능

# 📙 PART 05. ChatGPT 프로젝트로 전문가 되기 & 연구지원신청서 작성

### 1. ChatGPT 프로젝트 (Project) 기능 정의



* 하나의 명확한 목표를 중심으로 관련 대화 자료, 첨부 파일, 맞춤형 지침을 **한 공간에 모아서 점진적으로 작업을 관리하고 완성**해 나가는 장기 프로젝트용 작업 공간


* **핵심 이점:** 장기간에 걸친 복잡한 작업(보고서 작성, 팀 과제, 논문 집필)을 체계적이고 깔끔하게 정리할 수 있도록 지원



### 2. 프로젝트의 핵심 구성 요소



1. **프로젝트 이름:** "기후 변화 보고서 초안 작성"과 같이 작업을 식별할 수 있는 고유한 주제명


2. **파일 추가 (핵심):** 문서(Word, PDF, TXT)나 이미지를 드래그 앤 드롭으로 대량 업로드하여 AI가 프로젝트 내 모든 채팅에서 이 콘텐츠를 상시 참조하게 만듦


3. **지침 추가:** 결과물 유형, 목표 분량(예: A4 5~6장), 진행 단계 프로세스를 미리 정의 (현재 버전에서는 질의나 프롬프트에 직접 포함하여 수행)



### 3. 학술연구지원신청서 작성 7단계 및 원칙



#### 📋 연구제안서 작성 7단계 프로세스



1. 연구 주제 선정 및 문제 정의 -> 2. 선행연구 분석 (지식 공백 파악) -> 3. 연구 목적 및 가설 설정 -> 4. 연구 방법 및 절차 설계 -> 5. 연구 일정 및 예산 편성 (Gantt Chart) -> 6. 기대효과 및 활용 방안 수립 -> 7. 형식 점검 및 제출



#### ✍️ 교재 제시 논문/보고서 작성의 4대 원칙



* **명확성:** 추상적 표현 배제, 직접적이고 구체적인 문장 사용


* *(예)* "태도가 변화했다" -> "설문 결과 사후 태도가 긍정적으로 변화했다"




* **논리성:** 주장과 근거 간의 구조적 연결 (주제문 -> 설명 -> 근거/데이터 제시)


* **일관성:** 약어(LLM 등)는 최초 서술 시 풀어서 쓰고 이후 통일, 전문 용어와 기호의 문서 전체 일관성 유지


* **객관성:** 주관적 의견이나 1인칭 표현("우리는 생각한다") 배제, 수치와 자료 중심 문장 구성


* *(예)* "학생들이 좋아했다" -> "설문에서 82%가 긍정 응답을 보였다"





### 4. ChatGPT 답변의 한계와 인간의 보완 방법 (학술적 윤리)



* **답변 검토 및 수정:** AI가 생성한 내용이 연구자의 원래 의도와 일치하는지 내용의 탈락 여부 검토


* **구체적 정보 추가:** AI가 개략적으로 작성한 부분에 인간 연구자가 직접 수행할 구체적 연구 방법론을 채워 넣어 내용의 깊이를 확보


* **참고문헌 교정 (중요):** AI가 허구로 만들어낸 문헌이 있는지 **실제 존재 여부 및 저자, 출판 연도, DOI 정보를 직접 검증하고 수정**해야 함


* **학술적 윤리 4대 수칙:** 투명성 유지(AI 사용 사실 명시), 사실 확인(정확성 검증), 표절 방지(기존 연구와의 유사도 검사), **개인적 기여(연구자 본인의 독창적인 생각과 관점 추가)**
